Report and Insight¶

This report communicates the insights drawn from the WeRateDogs data wrangling project.

The insights drawn will be communicated also through visualization(s) that will be produced from the WeRateDogs wrangled data. The insights and visualizations that will be diplayed in this report will focused majorly on the doy_stages variable.

The four major dog stages indicated in the data set are; doggo, floofer, pupper, puppo and None for dogs with no specific stage. whwere their where dogs which fell into more than one dog stage. this we would discussed in details in this report.

Summary of Data Cleaning Procedure carried out in cleaning the WeRateDogs data set¶

The p1, p2 and jpg_url columns from the image_predictions.tsv were merged to the twitter_archive_enhanced data set on tweet_id. The user_id variable in the tweets_df data set was renamed to tweet_id to ease the merging of retweet_count and favorite_count from the tweets_id data set to the twitter_archive_enhanced data set. The doggo, floofer, pupper and puppo variable was melt into one variable called dog_stage. The data type of variables like timestamp, retweeted_status_timestamp where change from object to datetime. the in_reply_to_status_id, tweets_id, retweeted_status_user_id and in_reply_to_user_id where change to data type object using the .apply() method. This is because they are not numeric and are not needed to perform any calculations. The retweeted_status_id data was extracted from the expanded_urls variable using regular expresion. The rating_numerator and reating_denominator data were extracted from the text variable using regular expression regrex. he iloc() function was also used to fill the correct dog names in the name variable. the appropriate dog name was obtained from the text variable in the twitter_archive_enhanced.csv file and the cleaned data was stored to a csv file with name twitter_archive_master.csv
